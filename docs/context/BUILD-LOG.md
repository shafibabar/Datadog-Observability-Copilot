# BUILD-LOG.md — the journey (feeds the presentation meta-story)

Running narrative of how the solution came together: what was fast, what was hard, where AI accelerated, where human judgment was required.

## 2026-07-03 — Running it *without* an API key: the Claude Code way + Datadog tokens
- **Human judgment / direction:** The human has **no Anthropic API key** and directed that we run the reasoning through Claude Code itself (the same tool building this). Separately, they flagged that **Datadog now issues access tokens** rather than only key pairs, and asked for the README to cover both — plus how to *obtain* each credential, and how to run the metrics dashboard like the product.
- **AI caught the trap:** the ask *looked* like a docs-only edit, but the code supported neither path — writing those README steps first would have documented a lie. Flagged this up front and split it into **code-first, then docs**, per the operating model. Verified Datadog's current auth model by reading their docs (PATs authenticate standalone via `Authorization: Bearer`, legacy keys now "legacy") rather than guessing.
- **AI did (test-first):** added a second `LLMClient` behind the existing seam — `ClaudeCliClient` shelling out to `claude -p` headless (no key, reuses the Claude Code login) — with an `auto`/`cli`/`sdk` selector; added Datadog PAT (Bearer) support alongside the legacy pair; then rewrote the README (obtain-steps for both Claude paths + the PAT), `.env.example`, and the context files. **154/154 green, 99% cov, +11 specs, zero regressions.** The subprocess runner and CLI-availability check are injected, so the suite never spawns a real CLI or hits the network.
- **Meta-story value:** the clean `LLMClient` seam built weeks earlier meant a whole new *auth-free* way to run the product dropped in as one new class + a factory tweak — no rewrites. And the product now runs on the very tool that built it. The "looks like docs, is actually architecture" catch is exactly the plan-before-code discipline paying off.

## 2026-06-26 — Kickoff, planning, and foundation
- **Human judgment / direction:** The human (non-developer) supplied the charter and made the high-leverage calls — local run that can still connect to *real* production Datadog; chat-style UI with the **backend as the hero**; cost-conscious LLM use; and the firm constraint that **secrets are never committed** (loaded from an external file during the demo, Vault/Okta later).
- **AI accelerated:** Turned the charter into clarifying questions, a tech-stack recommendation with trade-offs, a phased roadmap, and the full project-memory structure — then scaffolded the foundation (secure config, FastAPI app, chat UI, context files) in one pass.
- **Key insight surfaced:** live Datadog vs. a reliable scripted demo are in tension → resolved cleanly with one `DataSource` interface and two adapters (live + replay). The architecture the charter asked for (clean seams) made this a non-issue.
- **Fast:** dependency choice, secret-handling design, and the chat UI skeleton.

## 2026-06-26 — Investigation Workspace (the architectural center of gravity)
- **Human judgment / direction:** Human chose to build the Workspace next (not run the foundation first) and approved the design in one pass — snapshot on every record, a confidence-history table, and Python-config (not table) sections.
- **AI did:** Before writing the schema, re-read the existing reasoning/telemetry models and **caught a design drift** — my initial proposal used a float confidence 0.0–1.0, but the codebase already standardised on a `Confidence` enum (low/medium/high). Aligned to the enum rather than introducing a second representation. Then wrote 14 specs red, implemented `app/workspace/{store,sections}.py` to green, full suite **57/57**, zero regressions.
- **Design payoff:** Because the reasoning engine already emits a structured `Investigation`, the Workspace is just *persist + history + index over that object* — clean seam, no reshaping. Sections are a registry (one-line append to add one), so personas and artifacts (next steps) transform over the same state.
- **Living-document property is real:** append-only snapshots (nothing overwritten), a `test_persists_across_store_instances` spec proves it's genuine SQLite state (not chat-derived), and confidence-over-time is queryable per hypothesis via a text-normalised key.
- **Next:** wire `/api/chat` to a per-session Workspace + ReasoningEngine; persona-rendered answers; "show me the evidence".

## 2026-06-26 — Chat endpoint wired (personas + the investigation loop)
- **Human judgment / direction:** Human said "go ahead and wire the chat endpoint."
- **AI did:** Added `app/personas.py` (registry of the 5 charter personas as config + a deterministic `render()`), `app/copilot.py` (`CopilotSession` joining DataSource + engine + Workspace), wired `/api/chat`, and updated the UI for evidence disclosure + persona-switch. 14 specs test-first; full suite **71/71**, zero regressions.
- **Two charter distinctions made concrete in the design — and they fell out cleanly:**
  - *ask vs. re-render.* A question runs a genuine investigation and **appends** a Workspace snapshot; switching persona (empty message) **re-renders the latest snapshot with no LLM call** — proving "facts don't change, only the lens" (§5.1) and saving spend. A test asserts `llm.calls` stays at 1 across a persona switch.
  - *evidence is always available.* Rather than a special "show evidence" round-trip, every reply ships its evidence catalog; the UI just discloses it. Matches §5.8/§6.7 "first-class, always-available" without extra endpoints.
- **Cost-conscious by construction:** persona rendering is deterministic composition over the structured Investigation — no second LLM pass for re-framing. The only spend is the one investigation per genuine question.
- **Graceful keyless degradation:** the session builds lazily; with no `ANTHROPIC_API_KEY` the chat returns a friendly "not configured" message instead of crashing, so the whole suite still runs offline and a non-dev can boot the app before placing keys.
- **Next:** the Incident Summary artifact (registry-driven transform over Workspace state), then README run instructions → Iteration 0 complete.

## 2026-06-26 — Incident Summary artifact (operational output)

- **The payoff of the shared Workspace is now visible:** the artifact is a *pure transform* over the same Investigation the chat already produced — a test asserts generating it makes **zero** new LLM calls. Same understanding, reshaped for incident responders, no manual rewriting (exactly §5.10's thesis).
- **Honest reasoning carried through:** the summary picks the *highest-confidence* hypothesis (a test proves the LOW-confidence decoy is excluded), states that confidence, derives severity from the timeline's peak event, and surfaces outstanding questions — it never overstates.
- **Iteration 0 status:** all build items complete; only the README run-instructions remain for the definition of done. The full demo narrative (dashboard/chat → health → changes → timeline → likely cause → evidence → persona switch → generated artifact, all from the living Workspace) is now achievable end-to-end with a key.

## 2026-06-26 — Refactor + hardening pass (autonomous, on a stronger model)
- **Human judgment / direction:** Human moved to a stronger model and asked for a full rescan: refactor where necessary, write/rewrite tests, run them — with autonomous permission to run commands.
- **AI did:** Read every source + test file, ran a coverage scan to find real gaps, then made *behavior-preserving* refactors (guarded by the existing green suite) and added targeted tests. Suite **82 → 98**, coverage to **99%**, zero regressions throughout.
- **Refactors (genuine, not churn):** (1) moved confidence/severity ordering onto the enums as `.rank`, deleting the duplicate rank-maps in `artifacts.py`; (2) extracted a shared `outstanding_questions()` helper so the Workspace section and the artifact can't drift; (3) type-safe `EventSource.SUPPORT` comparison instead of a magic string.
- **Highest-value new tests:** the **production factory** `build_default_session`/`_build_source` (replay/datadog/fallback/keyless — the wiring that actually runs live, previously untested → now 100%), a **security regression guard** that scans the raw workspace DB bytes for planted secrets (charter's non-negotiable: no secret ever persisted), engine rejection of non-object JSON, the `/api/artifact` 503-without-key path, and lazy session build+cache.
- **Restraint as a decision:** stopped at 99% — the last ~10 uncovered lines are defensive formatter/early-return branches; chasing them is the exact churn the charter warns against. Documented the call rather than padding the number.
- **Meta-story value:** demonstrates the AI using its own test/coverage tooling to find and close real gaps and to *justify stopping*, not just to write more code. The green suite + the security-guard test are objective proof the refactor changed structure without changing behavior.

## 2026-06-26 — Iteration 1: multiple conversations, memory, and a real UI
- **Human judgment / direction:** Human asked whether memory + multiple conversations existed (front and back), asked for real UI effort, and OK'd product token-spend. AI answered honestly (they did *not* exist — single global session, no message persistence, no dialog memory) and used AskUserQuestion to lock two genuine scope calls: **full recent-turn memory** and **sidebar + polish + live Workspace panel**.
- **AI did (test-first, 5 backend phases):** (1) `ReasoningEngine` now accepts bounded conversation `history`; (2) the store gained a `messages` table + `title`/`updated_at` + conversation listing/recency; (3) a type-dispatched `serialize_sections()` turns the heterogeneous Workspace sections into JSON for the panel; (4) `CopilotSession` became the conversation-aware **`Copilot`** service (per-`conversation_id`, persists turns, feeds memory, re-renders without re-reasoning); (5) a conversation-scoped API replaced the single `/api/chat`. Suite **98 → 113**, coverage held at **99%**, zero regressions.
- **Then the frontend (the visible payoff):** a 3-pane app — conversation **sidebar** (list/new/switch, last-opened remembered) · a restyled **chat** with markdown + per-reply "show me the evidence" · a collapsible **live Investigation Workspace panel** that re-renders the living document (health, timeline, hypotheses, evidence, confidence) after every turn, with a confidence/severity color vocabulary. Verified by booting the server (static assets + keyless degradation) and by asserting the exact JSON shapes the JS consumes via TestClient with a fake LLM.
- **Design wins that fell out of the existing seams:** the store was already per-workspace, so "multiple conversations" was almost free; the Workspace already *was* the memory substrate, so the panel is just a serializer over it; persona-switch re-render and artifact generation reuse the snapshot with **zero** extra LLM calls — so the only token spend is one investigation per real question (honoring "spend on product, save during demo").
- **Honest trade-off logged:** per-message evidence isn't persisted, so reloaded past bubbles drop their evidence disclosure; the always-current Workspace panel carries evidence instead (see DECISIONS). Frontend has no unit tests (no JS harness) — covered by boot + response-shape smoke tests instead.

## 2026-06-27 — Vibe-coding metrics subsystem (auditing the build itself)
- **Human judgment / direction:** Human asked for an ongoing metric-tracking system to audit the vibe-coding sessions, gave a schema + a 5-step plan (plan→approval, show populated file, TDD collector, TDD dashboard, host), and asked me to de-jumble "metrics vs analytics." This *is* the charter's meta-story (§7.2) made literal.
- **AI did, step-gated with approvals:** (1) planned the 3-layer design (data/collection/analytics), confirmed two forks via AskUserQuestion (FastAPI+Chart.js, Stop-hook). (2) **Reconstructed real history**: discovered Claude Code transcripts carry real per-turn `usage` tokens + timestamps; wrote a seeder that pulled exact tokens/timings from the two real session transcripts + line/file/test deltas from git + `TESTING.md`, producing 22 baseline `prompts.jsonl` entries (estimated fields flagged). (3) TDD collector (16 specs) — stdlib-only, Stop-hook driven, with per-turn git delta arithmetic that handles out-of-band commits and untracked files; **caught a real bug** (empty baseline counted the whole committed repo as "created") via its own integration test. (4) TDD dashboard data layer (14 specs) — tolerant loader + aggregations proven robust to malformed lines and future schema. (5) Hosted dashboard: FastAPI + a **dependency-free canvas charting module**.
- **Forced design change, handled cleanly:** the approved plan was vendored Chart.js, but the box has **no network** (download failed even outside the sandbox). Rather than a runtime CDN (breaks offline), I wrote a ~250-line zero-dependency canvas renderer (bar/stacked/line/doughnut + tooltips). Net result is *more* robust than the original plan: no library, no CDN, fully offline.
- **The hook proves itself:** after wiring `.claude/settings.json`, successive turns showed `prompts.jsonl` growing (22→23→24) on its own — live collection confirmed working end-to-end.
- **Meta-story payoff:** the project now measures its own build — tokens, durations, planning-vs-implementation split, cumulative green tests, lines/files churn, and docs/context growth — all from committed, human-readable data, with a local live dashboard. Zero new dependencies, no paid services.
